# Large_Language_Model_Deployment
项目概述
本项目为人工智能导论第三次课程作业，基于魔搭GPU Notebook平台，完成Qwen-7B-Chat、ChatGLM3-6B、Baichuan2-7B-Chat三款主流开源大语言模型的本地化部署。通过5道高难度中文歧义、嵌套逻辑语义试题，全方位测试三款模型的语义理解、逻辑推理、多义解析能力，并从显存占用、模型结构、算法亮点、实际性能、适用场景五个维度完成横向对比分析，总结不同大模型的差异化优势与适配场景。
一、实验部署环境
本次实验基于云端GPU开发环境完成部署，软硬件配置及依赖环境稳定，满足7B/6B量级大模型推理需求，具体参数如下：
- 运行实例：CPU Notebook
- 硬件配置：8vCPU、32GB RAM
- 系统镜像：ubuntu22.04-py311-torch2.3.1-1.37.1
- 核心依赖：Python 3.10 torch 2.3.0+cpu transformers 4.33.3 modelscope 1.9.5
二、部署模型列表
本次实验统一通过魔搭模型仓库git clone方式拉取模型权重，完成本地部署，涉及三款主流对话大模型：
1. 通义千问：Qwen-7B-Chat
2. 智谱清言：ChatGLM3-6B
3. 百川大模型：Baichuan2-7B-Chat
三、实验测试用例
本次采用5类经典中文语义难题，聚焦歧义理解、嵌套逻辑、多义词汇解析，测试模型中文理解核心能力：
1. 季节穿衣语义歧义：区分冬天、夏天“能穿多少穿多少”的反向语义差异
2. 双关语义理解：解析“谁都看不上”双重含义的幽默语境
3. 多层嵌套逻辑：拆解多重嵌套问句，定位核心未知主体
4. 人名语义歧义：区分叠词人名与动词语义，识别人物情感关系
5. 词汇多义解析：精准区分对话语境中“意思”的多重释义（心意、趣味、客套、含义等）
四、模型横向对比分析
4.1 基础资源占用对比
Qwen-7B-Chat依托GQA分组查询注意力机制，显存利用率最优，适配长文本扩展；ChatGLM3-6B参数体量最小，但双向注意力架构导致显存占用略高；Baichuan2-7B-Chat短文本推理显存效率优异，长文本场景显存消耗小幅上升。

4.3 模型结构与算法亮点
- Qwen-7B-Chat：基于Transformer-XL架构，搭载GQA分组查询注意力与动态NTK扩展技术，搭配RLAIF人工反馈强化学习，价值观对齐效果优异，支持对话压缩降本。
- ChatGLM3-6B：自研GLM通用语言架构，双向注意力+稀疏注意力设计，采用多阶段训练策略，支持INT4量化部署，硬件适配门槛低，对话交互体验流畅。
- Baichuan2-7B-Chat：RoPE增强Transformer架构，优化数学位置编码与滑动窗口注意力，动态批处理大幅提升推理吞吐量，内置安全护栏，结构化任务能力突出。
4.4 各模型优势场景总结
Qwen-7B-Chat 核心优势
- 长上下文文档分析、剧本创作等长文本任务
- 中文歧义句、双关语、多义词汇深度解析
- 开放域通用问答、复杂语义理解场景
ChatGLM3-6B 核心优势
- 多轮人机对话、智能客服、聊天机器人等交互场景
- 日常语义问答、轻量化文本生成任务
- 低硬件资源下的快速推理部署
Baichuan2-7B-Chat 核心优势
- 数学解题、数值计算、符号推理任务
- 代码生成、结构化数据输出场景
- 高并发批量推理、安全合规问答场景
五、实验结论
三款主流开源大模型的能力差异化显著，适配不同落地场景：Qwen-7B-Chat综合语义理解能力最强，擅长复杂中文歧义与长文本处理；ChatGLM3-6B对话交互体验最优，轻量化部署性价比高，适配日常人机交互；Baichuan2-7B-Chat在结构化、数理推理任务中优势明显，更适配技术类、计算类场景。本次实验完整完成模型部署、语义测试与横向对比，充分验证了不同架构大语言模型的性能特点与应用价值。
